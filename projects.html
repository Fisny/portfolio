IN-FLEET is a project proposal of mine, conceived as an alternative to the original version currently in production: (ORIGINAL IN-FLEET).

The development of this version aims to ensure complete autonomy in code management, avoiding dependency on DevExpress, which was considered limiting in this context.

We made significant improvements to the design and responsiveness while maintaining a visual familiarity with the version currently in production.

For the Front-End, ASP.NET was used in combination with jQuery and Bootstrap, while the Back-End was developed in C#.

Dapper was used as the ORM, and for managing complex data or advanced visualizations, Stored Procedures were implemented on SQL.

Although this project was not released into production, a demo video was created to showcase its features.

Project link: IN-FLEET

The main technologies used are:
  • ASP.NET
  • Jquery & Other JS libraries
  • Bootstrap
  • Log4Net
  • Dapper
  • C#
  • Microsoft SQL Server
  • Azure DevOps

IN-YACHT BLAZOR is a project designed to showcase the technological transition from the DevExpress stack (ASP.NET, .NET 4) to Blazor (.NET 8).

Unfortunately, despite considerable effort, the project was never released into production.

I chose to include it among my completed projects, as it represents a personal challenge in which I exceeded my own expectations.

Building on its predecessor, In-Yacht, which is already in production, I refined essential aspects, such as interface responsiveness and data processing speed.

Initially, I planned to use DAPPER as the ORM, as I am familiar with it, but to introduce an element of innovation, we decided to adopt Entity Framework.
